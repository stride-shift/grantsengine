#+TITLE: Product Context — d-lab Grant Engine
#+DATE: 2026-06-22

* Purpose
Present-state product context. Dormant/deferred features live in DEFERRED.org.
Read this before audits and before flagging code as an "issue" — several
properties below are intentional.

* What It Is
An internal tool for d-lab NPC's grant team to manage the funding pipeline:
scout opportunities, vet and shortlist them, draft AI-assisted proposals and
funder research, build budgets matched to d-lab's 8 programme types, track
deadlines, and store supporting documents. It is multi-tenant by organisation
slug but used primarily by one org (d-lab).

* Users & Roles
| Role | Scope | Can do |
|------|-------|--------|
| Super admin | All orgs | Provision orgs (create/delete) via the hidden =?superadmin= route, gated by =SUPER_ADMIN_KEY= |
| Org admin | One organisation | Manage members, org profile/onboarding, all grant + AI features |
| Member | One organisation | Work the pipeline: scout, vet, draft, budget, track deadlines |

Members sign in with **email + password**; the server resolves their org from a
globally-unique email (no org picking). Passwords are bcrypt-hashed; reset is
email-based via Resend. A legacy per-org member-picker login remains as a
migration fallback until every member has a backfilled email.

* Intentional Decisions (do not re-flag)
| Decision | Choice | Rationale |
|----------|--------|-----------|
| Monolithic App.jsx (~2400 lines) | Kept as one file | Views tightly coupled to shared grant state; splitting deferred (see DEFERRED.org) |
| prompts.js mostly LEGACY | Only scoutPrompt imported | App.jsx inlines its own prompts; LEGACY exports kept as reference, not dead by accident |
| grant-engine.monolith.jsx | Kept in src/ | Working single-file reference that runs in Claude artifacts — intentional, not stale |
| No TypeScript | Plain JSX | Project predates a TS migration; not a defect |
| No router | =view= state toggle | Single-page scope; router deferred |
| Inline styles, no CSS framework | Design tokens in theme.js | Enables standalone single-HTML demo build |
| CSP disabled in Helmet | =contentSecurityPolicy: false= | Vite injects inline scripts; intentional in app.js |
| ZAR currency, B-BBEE/SETA/NPO domain terms | Throughout | South African NPO context — not typos or i18n gaps |
| Email login primary; org auto-resolved | Single email+password screen, no org picker / admin-key field | Org resolved server-side from a global-unique email (=getOrgAndMemberByEmail=) |
| Member-picker now an unlinked dormant fallback | "Use member sign-in" link REMOVED from the email-login screen (2026-06-28); picker code retained | Email login is the sole linked path; picker reachable only on logout (OrgSelector) and still hosts the reset deep-link screens — see DEFERRED.org |
| Super-admin = real accounts + cross-org dashboard | =?superadmin= now requires a super-admin login (=super_admins= table, bcrypt, cross-org session token); dashboard lists every org with usage + activity and manages subscriptions (2026-06-29) | Replaces the old key-only provisioning. Legacy =SUPER_ADMIN_KEY=-gated org create/delete UI is superseded — see DEFERRED.org. Cross-tenant visibility is intentional (platform owner) |
| Subscription = manual tiers (no prices shown) | Free 1-week trial on sign-up → Monthly / Annual tiers (named only, no amounts in-app); super-admin sets plan/status by hand (no payment processor) | Expiry derived live from =trial_expires_at=; default is a non-blocking banner, with an opt-in per-org read-only lock (blocks edits + AI; viewing always works). Stripe + pricing deferred — see DEFERRED.org |
| Resources hub | Calendar / Documents / Funders / Archive are sub-tabs inside Resources (removed from sidebar; deep-link fallbacks kept); resource directory filtered by org type | =org_type= inferred from org details (=src/data/orgType.js=), overridable in Settings; corporate orgs no longer see NGO-only offers (2026-06-29) |
| Single "Log out" | Sidebar shows one "Log out" (real server logout); "Switch Organisation" removed | Avoided two near-identical buttons; switch-org returned to the picker without ending the session (2026-06-29) |
| Auto-submission is opt-in, never automatic | Playwright fill + submit are explicit, confirmed, form-only, service-gated | The one-click "Auto-draft & prepare" flow stops at a fully-prepared review; submitting to a funder is irreversible, so a human always confirms (=handleFinalSubmit= confirm dialog) |

NOTE: email login requires a member's email to be backfilled
(=server/backfill-emails.js=). The member-picker fallback is no longer linked from
the login screen (as of 2026-06-28); any member without a backfilled email must be
backfilled (or added by a director) rather than self-serving via the removed
"Use member sign-in" link.

NOTE: "Forgot password?" is the primary recovery path on the email-login screen. It
sends an email-only reset and shows the shared =CheckEmailScreen= ("Check your email
— if an account exists, we've sent a reset link"); the emailed link
(=?reset=<token>&slug=<org>=) opens the choose-new-password screen and auto-logs in.
