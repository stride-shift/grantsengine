* Purpose

The forward-looking counterpart to =PRODUCT_CONTEXT.org=. That file documents
what the product *is* (present-state); this file records what is deliberately
*not* finished, *not* decoupled, or *known-broken-but-not-yet-fixed* — and the
path to fix or reactivate each.

This is the one place allowed to talk about the future and about known defects.
It exists so audits don't re-flag intentional parking as a bug (and don't lose
a real gap as "probably intentional"). The cleanup effort is in *describe-don't-change*
mode: items here are *recorded*, not fixed, and parked logic questions are the
*owner's* call — they are written as open questions, never silently resolved.

The audit lifecycle axis (=AGENT_AUDIT_WORKFLOW.org=) uses the tags *Active /
Intentionally Dormant / Scheduled for Removal / Stale*. Each entry below carries
the tag an audit should apply so a finding here is reconciled, not re-raised.

=PRODUCT_CONTEXT.org= links here; it does not duplicate these entries. Code-level
detail lives in the relevant =quick_reference.org= / Level-1B docs (referenced per
row), not restated here.
